## sdivi-rust v0.2.0 — single assembly seam

This is the first minor bump on the 0.x line. It folds the WASM binding's
post-call `intent_divergence` override into the canonical `assemble_snapshot`
seam, so native and WASM callers now produce snapshots through the exact same
code path. Snapshot schema stays at `"1.0"`; existing snapshots are diffable
by 0.2.0 without conversion.

## Breaking change

`sdivi_snapshot::assemble_snapshot` (re-exported as
`sdivi_core::assemble_snapshot`) now takes `boundary_count: Option<usize>`
in place of `boundary_spec: Option<&sdivi_config::BoundarySpec>`.

- Callers that previously passed `Some(&spec)` should now pass
  `Some(spec.boundaries.len())`.
- Callers that passed `None` are unaffected.
- See [`MIGRATION_NOTES.md` § 0.1.x → 0.2.0](https://github.com/geoffgodwin/sdivi-rust/blob/main/MIGRATION_NOTES.md#01x--020)
  for the exact mechanical edit and rationale.

The change is recorded as ADL-10 in `.tekhton/ARCHITECTURE_LOG.md`. It removes
the only remaining edge between `sdivi-snapshot` and `sdivi-config`, and
eliminates the `intent_divergence` override seam that the WASM binding
maintained on top of the public API.

## Why

The WASM binding previously called `sdivi_core::assemble_snapshot` and then
mutated the returned `Snapshot`'s `intent_divergence` field by hand to
substitute its own `boundary_count` and `violation_count`. That created a
second assembly path with subtly different semantics — exactly the kind of
hidden seam that drifts. Threading `boundary_count` through the parameter
list collapses the two paths into one and removes a `sdivi-config` dependency
from the pure-compute crate at the same time.

## What did not change

- `snapshot_version` is still `"1.0"`. Snapshots written by 0.1.x load and
  diff cleanly under 0.2.0.
- No CLI flag, exit code, or `.sdivi/config.toml` key changed.
- The four divergence dimensions (pattern entropy, convention drift, coupling
  delta, boundary violation) and their threshold semantics are unchanged.
- WASM TypeScript types are tsify-derived from the same Rust types as before;
  the `assemble_snapshot` JS surface is unchanged for callers using the
  documented `WasmAssembleSnapshotInput` shape.

## Install

```sh
# crates.io
cargo install sdivi-cli

# pre-built binary (Linux x86_64 example)
curl -Lo sdivi https://github.com/geoffgodwin/sdivi-rust/releases/download/v0.2.0/sdivi-x86_64-unknown-linux-gnu
chmod +x sdivi && mv sdivi ~/.local/bin/

# WASM / npm
npm install @geoffgodwin/sdivi-wasm@0.2.0
```
