## sdivi-rust v0 — first public release

This is the first public release of sdivi-rust (pronounced *Stevie*),
the Rust reimplementation of the Structural Divergence Indexer. SDIVI is a
measurement instrument that tracks how fast the structure of a codebase is
changing.

It captures the structural fingerprint of a repository (dependency graph,
community partition, pattern catalog) into deterministic JSON snapshots and
reports four divergence metrics between consecutive snapshots: pattern
entropy rate, convention drift rate, coupling delta rate, and boundary
violation rate.

## Highlights

- **Single static binary** (`sdivi`) for Linux x86_64 + aarch64, macOS
  x86_64 + aarch64, and Windows x86_64. Stripped LTO release builds attached
  to this release.
- **Two-layer library shape.** `sdivi-core` is a pure-compute facade with no
  I/O, no clock, and no tree-sitter, suitable for embedding in WASM or
  agent runtimes. `sdivi-pipeline` is the FS orchestration layer used by
  the CLI and FS-based Rust embedders.
- **WASM bindings** as `@geoffgodwin/sdivi-wasm` on npm, with `tsify`-derived
  `.d.ts` for strict-TS callers. Every `compute_*` function is callable from
  TypeScript with the same determinism guarantees as native Rust.
- **Native Leiden community detection.** Verified within 1% modularity and
  ±10% community count of `leidenalg`. No FFI; the implementation is a
  native Rust port of Traag et al. 2019.
- **Six languages out of the box.** Rust, Python, TypeScript, JavaScript,
  Go, Java. Each language is gated behind a Cargo feature so binary-size
  sensitive consumers can strip what they do not need.
- **Snapshot schema is `1.0`.** Forward-additive: new fields land with
  `#[serde(default)]` and never bump the version within the 0.x line.
- **Boundary inference and ratification.** `sdivi boundaries infer`
  proposes module groupings from Leiden community history;
  `sdivi boundaries ratify` writes them to `.sdivi/boundaries.yaml`.
- **Per-category threshold overrides with mandatory `expires`.** Migration
  windows are first-class. After the date passes the override is ignored
  and defaults resume automatically.
- **Change-coupling analyser.** `change_coupling` is a snapshot field, and
  `boundaries.weighted_edges = true` lets co-change frequency pull
  highly-coupled files into the same Leiden community.
- **Snapshot at any commit.** `sdivi snapshot --commit REF` resolves the
  ref, extracts the tree via `git archive | tar`, and runs all five stages
  against it. Trend ordering follows commit-date, not invocation time.

## Public API surface

The following crates are stable and follow SemVer from this release on:

- [`sdivi-core`](https://docs.rs/sdivi-core) — pure-compute API
- [`sdivi-pipeline`](https://docs.rs/sdivi-pipeline) — FS-orchestrated pipeline
- [`sdivi-cli`](https://docs.rs/sdivi-cli) — the `sdivi` binary
- [`sdivi-config`](https://docs.rs/sdivi-config) — `Config` loader and types
- `sdivi-lang-{rust,python,typescript,javascript,go,java}` — language adapters
- `sdivi-graph`, `sdivi-detection`, `sdivi-patterns`, `sdivi-snapshot`,
  `sdivi-parsing` — inner crates (use only if you need a single stage)

## Install

```sh
# crates.io
cargo install sdivi-cli

# pre-built binary (Linux x86_64 example)
curl -Lo sdivi https://github.com/geoffgodwin/sdivi-rust/releases/download/v0.1.8/sdivi-x86_64-unknown-linux-gnu
chmod +x sdivi && mv sdivi ~/.local/bin/

# WASM / npm
npm install @geoffgodwin/sdivi-wasm
```

## Notes for adopters

- **No previous version is published.** This is `v0.1.8` because the
  workspace bumped through internal milestones during development, but no
  prior tag exists on crates.io or npm. There is no migration story from
  earlier sdivi-rust versions.
- **Migrating from sdi-py (the Python POC).** Snapshots are not backward
  compatible. See [`docs/migrating-from-sdi-py.md`](https://github.com/geoffgodwin/sdivi-rust/blob/main/docs/migrating-from-sdi-py.md).
- **Cross-platform determinism.** Aggregate metric equality is guaranteed.
  Bit-identity across platforms is not. See
  [`docs/determinism.md`](https://github.com/geoffgodwin/sdivi-rust/blob/main/docs/determinism.md).
