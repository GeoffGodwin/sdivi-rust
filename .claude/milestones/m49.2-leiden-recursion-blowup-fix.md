
#### Milestone 49.2: Leiden — Fix the `O(max_iterations^depth)` Recursion Blowup

<!-- milestone-meta
id: "49.2"
status: "pending"
-->

**Scope:** Eliminate the exponential-cost path in `leiden_recursive`
(`crates/sdivi-detection/src/leiden/mod.rs`) so `run_leiden` (and therefore
`Pipeline::snapshot` / `detect_boundaries`) terminates in bounded, near-linear
work on every input — including the pathological small `(graph, seed)` cases
captured in M49.1 — while preserving partition quality within the existing
`verify-leiden` tolerance (modularity within 1%, community count within ±10%)
and preserving bit-identical determinism for the same seed. Un-ignore and pass
the M49.1 regression test.

**Why this milestone exists:** Profiling and code reading trace an intermittent
multi-minute hang on an **8-node** graph to the structure of `leiden_recursive`:

```text
for _iter in 0..max_iter {                 // max_iterations = 100
    let moved = local_move_phase(...);     // bounded, single sweep
    if !moved { break; }
    let refined = refine_partition(...);   // bounded (MAX_REFINE_ITER = 10)
    let agg = aggregate_network(...);
    if agg.n >= graph.n { break; }                 // no-compression guard
    if agg.n as f64 > graph.n*(1.0-min_ratio) { break; }  // M28 compression cutoff
    let agg_result = leiden_recursive(&agg, ..., depth + 1);  // <-- full descent EVERY iter
    partition = flatten_partition(...);
}
```

Every one of the up-to-`max_iter` outer iterations performs a **full recursive
descent** on the aggregated graph. For inputs where local moves keep firing
(`moved == true`) and compression stays at/above `min_compression_ratio` each
pass, the cost compounds multiplicatively to `O(max_iterations^depth)` — on the
order of `100^8 ≈ 10^16` for an 8-node graph. The M28 recursion-depth cap
(`leiden_max_recursion_depth = 32`) and compression-ratio cutoff bound the
**depth** of any single descent but not the **number of descents** the outer loop
spawns, so they do not contain the blowup. Most inputs converge in 1–3 outer
iterations and never expose it; the M49.1 case does not.

**Theoretical basis:** Traag et al. 2019, *From Louvain to Leiden*, Algorithm 1.
The canonical structure recurses on the aggregate network **once per level** —
the per-level loop alternates local-move + aggregate until the partition is
stable, and the aggregate is descended into a single time, not re-descended on
every local-move sweep. The fix should bring the implementation in line with this:
the recursive call belongs to the level transition, not inside the
move-until-stable loop. Equivalent, lower-risk containments are acceptable if they
provably bound total work (see Deliverables).

**Deliverables (pick the minimal correct fix; first is preferred):**

- **Restructure the level loop** so `leiden_recursive` descends into the aggregate
  at most once per level: run local-move (and refinement) to convergence at the
  current level first, then aggregate and make a single recursive call — instead of
  recursing inside each outer iteration. Verify against `verify-leiden` that quality
  is unchanged within tolerance.
- **If full restructure is too invasive for one milestone,** add a provable global
  bound: a shared work/visit budget (e.g. total node-moves or total recursive calls)
  threaded through `leiden_recursive`, plus a convergence guard that breaks the outer
  loop when the partition's quality stops strictly improving (ΔQ ≤ ε) rather than only
  when `moved == false`. Either must guarantee total work is polynomial in `n` and
  independent of the multiplicative explosion.
- Un-ignore the M49.1 regression test and make it pass (terminates fast).
- Update `docs/determinism.md` / `MIGRATION_NOTES.md` only if the fix changes any
  snapshot output (it should not for the common case; if seed-stable assignments
  shift for some inputs, note it as a determinism-preserving but value-changing fix).

**Migration Impact:** Target is **none** to schema or public API; `snapshot_version`
stays `"1.0"`. If the restructure changes cluster assignments for some inputs (it
may, since the current loop's redundant descents can alter the final partition),
the change must remain deterministic for a fixed seed, stay within `verify-leiden`
tolerance, and be called out in `CHANGELOG.md` + `MIGRATION_NOTES.md` as a
quality-preserving partition change (same class as M28's ≤1% modularity note).

**Files to create or modify:**

- `crates/sdivi-detection/src/leiden/mod.rs` — `leiden_recursive` loop structure / work bound.
- Possibly `crates/sdivi-detection/src/partition.rs` (if a budget field is added to `LeidenConfig`).
- `crates/sdivi-detection/tests/refinement.rs` / `tests/leiden_termination.rs` — un-ignore the M49.1 guard.

**Acceptance criteria:**

- The M49.1 minimal `(graph, seed)` case (and a sweep over `n in 3..=8`, all
  `seed in 0..=255`, dense and sparse edge sets) completes `run_leiden` in
  bounded time (well under 1s each); no input hangs.
- `cargo test --workspace` and `cargo test --workspace --features verify-leiden`
  are green on the full CI matrix, with no `cancelled`/timeout legs.
- `verify-leiden` quality gates hold (modularity within 1%, community count ±10%).
- Determinism preserved: same repo state + same seed ⇒ bit-identical `Snapshot`
  (a different-but-stable output than pre-fix is acceptable if documented).

**Tests:** Reuse the M49.1 regression + the existing `refinement.rs` proptests
(now bounded by the M49.1 timeout). Add a brute-force termination test sweeping
small graphs × all seeds under a per-case wall bound. Run `leiden_quality.rs`
(`verify-leiden`).

**Watch For:** Do not simply lower `max_iterations` or raise
`min_compression_ratio` to dodge the blowup — that degrades partition quality and
only narrows the pathological window. The fix must bound total work structurally.
Preserve the M17 self-loop and M28 induced-subgraph-refinement behaviour. Keep RNG
usage seed-deterministic (Rule 18) — any change to move ordering or recursion count
changes the RNG draw sequence and thus the output; re-baseline determinism fixtures
intentionally, not accidentally.

**Seeds Forward:** With termination bounded, consider a `benches/` criterion case on
a known-pathological graph to catch future regressions in the performance contract.
